---
description: "How to use the GSD (Get Shit Done) workflow"
---

1. /gsd:new-project - Initialize a new project/feature.
2. /gsd:discuss-phase - Define phase-specific requirements.
3. /gsd:plan-phase - Generate execution plans.
4. /gsd:execute-phase - Implement code with verification.
5. /gsd:validate-phase - Verify phase integrity.
6. /gsd:verify-work - Final manual verification.
7. /gsd:audit-milestone - Check overall project status.
8. /gsd:complete-milestone - Wrap up current work.
9. /gsd:new-milestone - Start next major milestone.
